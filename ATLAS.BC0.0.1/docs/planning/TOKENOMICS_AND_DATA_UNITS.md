# ATLAS Tokenomics & Data Unit Mechanics

This document outlines the core economic and interaction mechanics of the ATLAS blockchain, specifically differentiating between the native coin (TCOIN) and the interaction-based energy system known as "Data Units" (DU).

## 1. Core Concepts

### A. The Native Coin (TCOIN)
- **Hard Cap:** 1 Billion Units.
- **Purpose:** Mainstore of value, network security (staking/validators), and medium of exchange.
- **Generation:** Minted through block rewards (validators) or initial genesis allocation. Cannot be generated out of thin air by users.
- **Holding Perk (Passive Generation):** Holding TCOIN provides a passive generation rate of "Data Units". The more TCOIN a user holds or stakes, the higher their generation rate of Data Units.

### B. Data Units (DU) - "The Energy of Interactions"
- **Purpose:** Represents the cost and energy (hardware/software/cognitive) required to sustain an interaction within the system. It is the fuel for social mechanics.
- **Nature:** DUs are fluid, continuously generated and consumed by user activity.
- **Not a Token:** Unlike TCOIN, DUs are an environmental metric, a sub-layer fuel for the platform's social physics, not a directly tradable speculative asset on mainnet exchanges. 

## 2. Interaction Mechanics: The Flow of Data

### Information Flow
Data Units (DUs) define the "weight", "context", and "energy" of an interaction. Every user and object (post, video, item) is tagged with a "Thread" (descriptive categories, interests, occupations).

### The Origin of Data Units (Genesis)
When a new or recurrent user enters the system, they are granted an initial allocation of DUs. This represents their unique digital footprint ("Hello, I'm here today, John Doe, with these occupations and interests"). This base data holds inherent value in the ATLAS ecosystem.

### Scenario Example:
1. **The Object (Floating Element):** A user submits a video to the feed. *Crucially, submitting an object does not burn DUs.* Instead, it takes the user's current DUs and tags them to the object (User's context + Object's threads). The object becomes a "floating element" in the network that holds these tagged DUs for external interactions.
2. **The Encounter:** Another user explores the feed and encounters the video.
3. **The Interaction (Fusion):** The exploring user interacts with the object (upvotes, buys, pins, scrolls). 
4. **The Burn & Metric Push:** The interaction causes a "fusion" of the exploring user's DUs and the object's held DUs. The resulting tagged data packet is pushed into the ATLAS metric system and then *burned*.
5. **Dynamic Market (Macroeconomic Model):** The metric system aggregates these burned fusion packets to generate "Conclusions" across the network (e.g., "Users with X characteristics buy more food," or "The bathing suit sector of the clothing industry has seen an uprise"). This creates a Dynamic Market resembling reality. It captures the value generated by data trends (consumer shifts, behavioral correlations) and funnels this macro-valuation straight back into the TCOIN utility, rather than selling it off to centralized tech companies. 

## 3. The Lifecycle of an Object

### Creation (The Tagging phase)
- **Submission:** Placing an object on the platform attaches the user's specific weighted Data Units (their traits, status, and tags) and bonds them to the object's threads. The object is now a carrier of potential energy.

### Sustenance, Fusion & Value
- **Interaction Pool:** As external users interact with the object, fusion occurs. The object converts these interactions into pushed metrics and burned DUs.
- **Extraction (Value Conversion):** Object authors can extract value from highly-interacted objects based on the amount of fusion metrics generated. Highly valuable networks of burned DUs can be redeemed for rewards from the TCOIN emission pool.

## 4. The Loop: Tokenomics & Mechanics

1. **Hold TCOIN:** Users hold native coins.
2. **Generate DUs:** Holding TCOIN passively generates DUs over time.
3. **Interact:** Users spend DUs to post, upvote, and explore, generating rich threaded metadata for the network.
4. **Accumulate:** High-quality objects accumulate DUs from user interactions.
5. **Extract:** Authors extract value from successful objects, linking the social interaction layer back to the main economic tokenomics layer.

## 5. Summary
- **TCOIN:** Static, capped, store of value.
- **Data Units (DU):** Dynamic, fluid, generated by holding tokens and user interactions, acting as the fuel for information propagation and content valuation.
